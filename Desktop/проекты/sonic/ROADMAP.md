# PS Club — Roadmap

> Система управления игровым клубом PlayStation  
> Stack: Next.js 14 · Supabase · Vercel  
> Тема: тёмная (#1a1a2e + #352F73)

---

## Фаза 0 — Фундамент
*Настройка проекта, база, авторизация*

- [ ] Инициализировать Next.js 14 (App Router, TypeScript, Tailwind)
- [ ] Подключить Supabase (создать проект, получить ключи)
- [ ] Настроить `.env.local` и Supabase client
- [ ] Создать схему БД: `clubs`, `users`, `rooms`, `sessions`, `orders`, `bookings`
- [ ] Настроить Row-Level Security (RLS) для ролей `owner` / `admin`
- [ ] Создать страницу `/login` (Supabase Auth, email + password)
- [ ] Middleware для защиты роутов + редирект по роли
- [ ] Сидировать тестовые данные (2 клуба, комнаты, тестовые юзеры)
- [ ] Деплой на Vercel, проверить env variables

---

## Фаза 1 — Дашборд администратора
*Главная рабочая страница*

- [ ] Layout для `/dashboard` (сайдбар, хедер с названием клуба)
- [ ] Grid карточек комнат (`/dashboard/rooms`)
- [ ] Карточка комнаты: статус (свободна/занята/забронирована), цвет-индикатор
- [ ] Live таймер на карточке (client-side, обновляется каждую секунду)
- [ ] Supabase Realtime — подписка на изменения `rooms` и `sessions`
- [ ] Кнопка "Начать сессию" → модал (имя клиента) → старт одним тапом
- [ ] Кнопка "Завершить сессию" → авторасчёт суммы → показать инвойс
- [ ] Пауза / Возобновление сессии
- [ ] 10-секундный undo toast после завершения сессии

---

## Фаза 2 — Заказы и оплата
*Еда, напитки, итоговая сумма*

- [ ] Страница / модал добавления заказа к активной сессии
- [ ] Справочник товаров (item_name + price) — редактируемый
- [ ] Закреплённые (часто используемые) товары вверху списка
- [ ] Авторасчёт: время × тариф + сумма заказов = итого
- [ ] Инвойс-экран с итогом (для показа клиенту)
- [ ] История сессий с фильтрацией по дате и комнате

---

## Фаза 3 — Бронирования
*Запись клиентов заранее*

- [ ] Страница `/dashboard/bookings` — список броней
- [ ] Форма создания брони (дата, время, комната, клиент, телефон)
- [ ] Проверка конфликтов при бронировании
- [ ] Статусы брони: активна / завершена / отменена
- [ ] Бейдж на карточке комнаты если есть ближайшая бронь
- [ ] Алерт/пульсация карточки за 15 минут до окончания сессии
- [ ] Кнопка "Заселить" — конвертирует бронь в активную сессию

---

## Фаза 4 — Панель владельца
*Обзор двух клубов*

- [ ] Layout `/owner` (отдельный от `/dashboard`)
- [ ] Страница `/owner/clubs` — два клуба side-by-side, live статусы комнат
- [ ] Выручка: сегодня / неделя / месяц по каждому клубу
- [ ] Количество сессий, средняя длительность
- [ ] Страница `/owner/analytics` — графики (recharts / chart.js)
- [ ] Пиковые часы — тепловая карта по дням/часам
- [ ] Экспорт отчёта (CSV или PDF) за период

---

## Фаза 5 — UX-фишки и автоматизации
*Вещи, которые делают продукт "вау"*

- [ ] Сводка смены — кнопка в конце дня генерирует дневной отчёт
- [ ] Автозавершение сессии — таймер обратного отсчёта если бронь на фиксированное время
- [ ] Управление тарифами прямо из интерфейса (без БД-редактора)
- [ ] Тёмная тема — полный UI-полишинг (#1a1a2e + #352F73)
- [ ] Адаптивность — работает на планшете (основное устройство админа)
- [ ] Настройки клуба: название, адрес, тариф по умолчанию

---

## Фаза 6 — Масштабирование (пост-MVP)
*Идеи на будущее*

- [ ] Таблица `clients` — история визитов, бонусы, скидки
- [ ] Система лояльности (N визитов → скидка X%)
- [ ] Добавление третьего и последующих клубов без переписывания
- [ ] Push-уведомления для клиентов (бронь подтверждена)
- [ ] Мобильное приложение (React Native + тот же Supabase)
- [ ] Дашборд для клиентов (проверить свою бронь онлайн)
