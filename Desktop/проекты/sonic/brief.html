Основной функционал
⏱️

Учёт времени

Старт / пауза / стоп. Авторасчёт стоимости. 10-секундный undo после завершения.

💰

Расчёт оплаты

Время × тариф + заказы = итог. Тариф меняется через интерфейс без БД-редактора.

📅

Бронирования

Запись с проверкой конфликтов. Один тап «Заселить» → сессия. Пульсация за 15 мин до конца.

🍔

Заказы

Еда и напитки к сессии. Закреплённые популярные позиции. Добавляются одним тапом.

📊

Аналитика

Для владельца: выручка по клубам, пиковые часы, средняя длительность сессии.

🔄

Realtime

Supabase Realtime — изменения мгновенно видны без перезагрузки страницы.

Роли и доступ
Owner
Видит оба клуба: live статусы комнат, выручку, аналитику. Может смотреть, но не управляет сессиями напрямую.
Admin
Видит только свой клуб. Полный контроль: комнаты, сессии, бронирования, заказы. RLS в Supabase гарантирует изоляцию.
Схема базы данных
clubs         id · name · address · hourly_rate
  └─ rooms     id · club_id · name · type · status · hourly_rate
       └─ sessions id · room_id · club_id · client_name
                     started_at · ended_at · paused_duration
                     total_amount · status
            └─ orders  id · session_id · item_name · price · qty
  └─ bookings  id · club_id · room_id · client_name
                  phone · starts_at · ends_at · status
users         id · email · role(owner|admin) · club_id
